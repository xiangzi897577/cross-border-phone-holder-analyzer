# AGENTS.md

## 项目名称

跨境电商手机支架选品分析平台
Cross-border Phone Holder Product Analyzer

## 项目定位

本项目是一个用于前端暑期实习简历展示的前后端分离项目。

项目聚焦“手机支架”这个跨境电商轻小件品类，围绕模拟 Amazon 候选商品与 1688 货源数据，完成商品利润测算、竞争强度评估、物流成本分析、风险等级判断、推荐评分、候选池管理和图表分析等功能。

当前项目 Day 1-32 的 mock 数据版核心业务已经基本跑通，后续进入项目增强、部署上线、UI 优化、README、截图和简历化阶段。

本项目的最终目标是：

* 可以在线演示
* 可以写进简历
* 可以讲清楚前后端数据流
* 可以讲清楚业务逻辑
* 可以体现 React + Node.js 全栈项目能力

## 开发边界原则

从第六阶段开始，本项目不再默认严格禁止数据库、第三方库、UI 组件库、真实 API、部署等扩展能力。

但是必须遵守一个核心原则：

**只执行用户当前明确分发的任务。用户没有明确要求的功能，不要擅自添加。**

例如：

* 用户要求 UI 优化时，可以优化 UI。
* 用户要求使用 UI 组件库时，可以评估并引入。
* 用户要求部署上线时，可以进行部署适配。
* 用户要求数据库改造时，可以设计并接入数据库。
* 用户要求接入 1688 API 时，可以进行 API 接入或适配层设计。
* 用户要求接入 Amazon API 时，可以进行 API 接入或方案设计。
* 用户没有明确要求时，不要主动加入数据库、登录、鉴权、真实 API、复杂状态管理或大规模重构。

## 代码风格

1. 文件名使用清晰语义。
2. 组件名使用大驼峰命名。
3. 工具函数放到 utils 目录。
4. API 请求函数放到 services 目录。
5. 后端路由放到 routes 目录。
6. 不在页面组件里堆太多业务计算逻辑。
7. 能抽离的计算逻辑尽量放到 utils 中。
8. CSS 类名要语义化。
9. 组件职责要清晰。
10. 页面组件负责组织数据和页面结构。
11. 展示组件负责 UI 展示。
12. API 层负责请求封装。
13. 后端 utils 负责业务计算和文件读写等通用逻辑。

## 部署说明

当前项目是前后端分离结构：

* 前端适合部署到 Vercel
* 后端适合部署到 Render、Railway、Koyeb 等平台

部署时需要注意：

1. 前端 API 地址不要写死 localhost。
2. 前端应支持通过环境变量配置线上后端地址。
3. 后端应使用 process.env.PORT。
4. 后端 CORS 需要允许线上前端域名访问。
5. React Router 刷新页面时可能需要前端平台 rewrite 配置。
6. 当前 JSON 文件存储在线上不是生产级持久化方案。
7. 如果需要长期保存用户数据，应迁移到数据库。
